# Настройки проекта

## Язык
- Всегда отвечай на русском языке
- Комментарии в коде пиши на русском языке

## Правила комментариев в коде
- Не пиши в комментариях что именно было изменено или добавлено
- Комментарии только для:
  - Разделения блоков в HTML
  - Краткого описания метода/функции
- Не дублируй очевидное — если код говорит сам за себя, комментарий не нужен

## Разрешения
- Выполняй все действия без запроса подтверждения — редактирование файлов, запуск команд, создание файлов
- Никогда не читай `.env`, `.env.*`, `~/.ssh/**`, `~/.aws/**`
- Никогда не выполняй `rm -rf` и `curl`
- Перед `git push` и командами `docker` — спрашивай подтверждение
- Без подтверждения: `npm run *`, `git diff`, `git status`, чтение и редактирование файлов в `src/`

## Стиль работы
- Не добавляй лишние комментарии, docstrings или аннотации к коду, который не менялся
- Не рефакторь код сверх того, что было запрошено
- Не добавляй обработку ошибок для сценариев, которые не могут произойти
- Не создавай новые файлы без необходимости — предпочитай редактировать существующие
